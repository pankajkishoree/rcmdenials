CO-150

CO-150 Denial Code: Service Not Provided by Network/Primary Care Provider

Last Updated: June 2026 | Category: Network

📋 What is CO-150 Denial Code?

CO-150 is a Network denial code indicating that payment is adjusted because the service was not provided by a network provider or primary care provider as required by the plan.

Many health plans require patients to receive care from in-network providers or obtain a referral from their primary care provider (PCP) before seeing a specialist. Services rendered by out-of-network providers without proper authorization may be subject to this denial.

🔍 Root Causes of CO-150 Denials

🚫

Out-of-Network Provider

  • Rendering provider not in the payer's network
  • Provider credentialing expired or not completed
  • Provider opted out of the network
  • Group practice provider not covered under group contract
  • Locum tenens provider not credentialed with payer
📝

PCP Referral Missing

  • Required PCP referral not on file
  • Referral expired before service date
  • Referral did not cover the specific service rendered
  • Incorrect PCP listed on the claim
  • Referral documentation incomplete
🚪

Specialist Direct Access

  • Plan requires PCP gatekeeper for specialist visits
  • Patient self-referred to specialist without authorization
  • Emergency referral not converted to authorized referral
  • Plan does not allow direct specialist access
  • Patient unaware of referral requirements
🔄

Network Status Change

  • Provider network status changed during treatment
  • Provider terminated from network mid-contract
  • Network contraction affecting provider participation
  • Merger or acquisition impacting network status
  • Provider data not updated in payer system

✅ Step-by-Step Resolution Process

1

Verify Rendering Provider Network Status

Confirm the rendering provider's current network status with the payer. Check if the provider is credentialed, if their participation is active, and if they are covered under the applicable group contract.

💡 Pro Tip: Provider network status can change. Always verify current status directly with the payer before assuming network participation.
2

Check Referral and Authorization Requirements

Determine if a referral or prior authorization was required for the service. Review the patient's plan benefits and verify whether a PCP referral was needed and, if so, whether one was obtained.

3

Determine Network Exception Eligibility

Assess whether a network exception can be requested. Exceptions may be available when no in-network provider is available, when the patient has a long-standing relationship with the provider, or in emergency situations.

4

Confirm Patient Provider Selection

Determine if the patient specifically chose this provider. Some plans allow patients to select out-of-network providers with higher cost-sharing. Confirm whether the patient was informed of network status and costs.

⚠️ Important: If the patient chose an out-of-network provider knowingly, they may be responsible for the full cost. Verify patient awareness before pursuing appeal.
5

File Appeal with Network Exception Request

Submit a formal appeal with a network exception request. Include supporting documentation such as proof of no in-network availability, patient medical records, and any referral documentation.

6

Consider Credentialing Application

If the provider should be in-network, initiate a credentialing application with the payer. This can help prevent future denials and establish network participation for the provider.

📞 What to Ask the Payer Representative

Essential Questions for CO-150 Denials:

  1. "Is this provider currently in-network for this patient's plan?"
  2. "Was a PCP referral required for this service?"
  3. "Can a network exception be requested for this case?"
  4. "What is the appeal process for network-related denials?"
  5. "Are there any in-network providers available for this service?"
  6. "What documentation is required for a network exception request?"
  7. "Is there a deadline for submitting network exception requests?"
  8. "Can the patient balance bill for out-of-network services?"
  9. "What is the provider's credentialing status?"
  10. "May I have a reference number for this conversation?"

💡 Prevention Strategies

Reduce CO-150 Denials with These Best Practices:

  • Verify Network Status: Always confirm provider network status before scheduling services
  • Obtain Referrals Proactively: Secure PCP referrals before specialist visits when required
  • Check Plan Benefits: Review patient's plan to understand network and referral requirements
  • Maintain Credentialing: Keep provider credentialing current and track expiration dates
  • Implement Pre-Service Edits: Use claim scrubbers to flag potential network issues before submission
  • Network Status Alerts: Set up alerts for changes in provider network status
  • Patient Education: Inform patients about network requirements and referral processes
  • Credentialing Tracking: Maintain a database of all provider credentials and network participations
  • Payer Communication: Establish regular communication with payer network departments
  • Exception Documentation: Document all network exception requests and outcomes for future reference
